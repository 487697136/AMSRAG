# 知识库C 测试问题与答案生成提示词

> 使用方法：将【提示词正文】复制到AI对话中，并附上你已经生成的8个文档的全部内容。
> AI会根据文档的实际内容生成精确的问题和参考答案。

---

## 提示词正文

```
你是一个RAG（检索增强生成）系统的测试工程师。我将提供一组企业知识库文档（共8篇），请你根据文档的实际内容，为我设计一套完整的测试问题集。

【背景说明】
这些文档属于一个虚构公司"星辰智联科技集团"的内部知识库。我需要用这些问题来测试和演示一个RAG系统的效果。该系统具备以下核心能力：
1. 查询复杂度自动分类（zero_hop / one_hop / multi_hop）
2. 多种检索模式（仅模型回答、本地图检索local、朴素向量检索naive、关键词检索bm25、全局检索global）
3. 知识图谱构建与可视化
4. CA-RRF多源融合检索

【问题设计要求】

请严格按照以下6个类别，每个类别生成5个问题，共30个问题。
每个问题必须包含以下字段：

- 编号
- 问题文本
- 复杂度类别（zero_hop / one_hop / multi_hop）
- 预期触发的检索模式（llm_only / local / naive / bm25 / global）
- 需要的来源文档（列出文档编号）
- 参考答案（基于文档内容的准确答案，200-400字）
- 演示价值说明（一句话说明这个问题能展示系统的什么能力）

==========================================================
类别一：简单事实查询（zero_hop → 预期触发 llm_only）
==========================================================

设计5个最简单的事实性问题，特点：
- 答案是一个明确的事实（人名、日期、数字等）
- 只需要单一信息点，不需要任何推理
- 用于演示"仅模型模式"的对比 —— 不用知识库时LLM完全答不出来

问题风格示例（仅供参考，请根据实际文档内容设计）：
- "星辰智联的创始人是谁？"
- "公司是哪一年成立的？"

【重要】这类问题的演示策略是：先用"仅模型"模式提问（LLM无法回答），再切换到使用知识库模式（精准回答），形成鲜明对比。

==========================================================
类别二：单实体检索查询（one_hop → 预期触发 local 或 naive）
==========================================================

设计5个需要检索单个文档中特定段落才能回答的问题，特点：
- 围绕一个具体实体展开（某个人、某个产品、某个部门）
- 需要从文档中找到相关段落提取信息
- 答案需要2-3个信息点的整合，但都在同一文档内

问题风格示例：
- "韩子轩在公司中担任什么职位，他的主要职责是什么？"
- "'天枢'智能对话平台的核心技术架构是什么？"

==========================================================
类别三：跨文档关联查询（one_hop → 预期触发 local + bm25）
==========================================================

设计5个需要从2个不同文档中提取信息才能完整回答的问题，特点：
- 答案分散在2个文档中
- 需要将两个文档的信息拼合才能形成完整回答
- 用于展示系统的多源检索和信息融合能力

问题风格示例：
- "楚云帆负责的'银河'升级计划具体包含哪些目标，该计划在2025年规划中的优先级如何？"
  （需要文档4的项目详情 + 文档5的年度规划信息）

==========================================================
类别四：多跳推理查询（multi_hop → 预期触发 local + global）
==========================================================

设计5个需要跨3个及以上文档进行多步推理的问题，特点：
- 需要先找到A，从A找到B，再从B找到C
- 答案无法在任何单一文档中直接找到
- 需要系统进行实体关联和推理
- 这是系统的核心差异化能力展示

问题风格示例：
- "负责'星火'大模型项目中数据准备工作的团队成员，同时还参与了哪些其他产品的研发？"
  （文档8→找到具体人名→文档2→找到其他职责→文档3→找到参与的产品）

==========================================================
类别五：全局分析查询（multi_hop → 预期触发 global）
==========================================================

设计5个需要对整个知识库进行全局性分析才能回答的问题，特点：
- 没有单一确切答案，需要综合归纳
- 涉及趋势分析、战略评价、对比总结等高层次理解
- 用于展示global模式的社区报告和全局分析能力

问题风格示例：
- "从组织架构、产品布局和战略规划来看，星辰智联的核心竞争力体现在哪些方面？"
- "星辰智联在技术研发和商业落地之间是如何平衡的？"

==========================================================
类别六：精确召回与对比查询（one_hop → 预期触发 bm25 + local）
==========================================================

设计5个需要精确匹配特定条款、数字、规定的问题，特点：
- 涉及具体的制度条款、数字指标、时间节点
- 关键词精确匹配（BM25）比纯语义检索更有效
- 用于展示混合检索（BM25 + 向量）的互补优势

问题风格示例：
- "员工年假天数是如何根据工龄划分的？具体审批流程是什么？"
- "'朱雀'工业质检系统的缺陷识别准确率达到了多少？"

==========================================================
输出格式要求
==========================================================

请按以下格式输出每个问题：

---
【Q01】类别一 · 简单事实查询
问题：[问题文本]
复杂度：zero_hop
预期模式：llm_only（对比演示用）
来源文档：文档1
参考答案：[基于文档内容的准确答案]
演示价值：[展示RAG的什么能力]
---

请确保：
1. 所有参考答案必须严格基于提供的文档内容，不得编造文档中没有的信息
2. 来源文档标注必须准确，指明答案信息具体来自哪个/哪些文档
3. multi_hop类问题必须清晰标注推理链条（A→B→C）
4. 30个问题覆盖尽可能多的实体（人物、产品、项目、制度等）
5. 问题之间不要重复考察同一个知识点

现在请根据以下文档内容生成30个测试问题：

[请在此处粘贴你的8个文档全文]
```

## 补充：演示专用精选问题生成提示词

> 如果你需要从30题中精选5-8个用于现场演示的问题，可以在生成30题之后追加以下提示：

```
从上面30个问题中，帮我挑选出最适合现场演示的8个问题，要求：

1. 覆盖全部6个类别，每个类别至少选1个
2. 优先选择"对比效果最鲜明"的问题（有知识库vs无知识库差异最大的）
3. 优先选择"涉及实体最多"的问题（图谱可视化效果最好的）
4. 按照演示顺序排列：从简单到复杂，形成递进式的展示节奏

对每个精选问题，补充以下演示指导：
- 演示话术：向评委介绍这个问题时怎么说（1-2句）
- 操作步骤：在系统界面上的操作流程
- 预期亮点：评委会注意到的关键效果
- 对比策略：是否需要先用"仅模型"模式做对比
```

## 补充：红楼梦知识库的测试问题生成提示词

> 这是针对知识库A（红楼梦）的测试问题生成提示词，可单独使用。

```
请为《红楼梦》知识库设计20个测试问题，分为以下4个类别，每类5个。

设计原则：选择LLM可能答错细节、给出模糊回答、或无法精确溯源的问题。
避免设计LLM能轻松正确回答的常识性问题（如"贾宝玉和林黛玉什么关系"）。

==========================================================
类别一：细节精确查询（LLM容易出错的具体细节）
==========================================================
设计LLM很可能答错或模糊回答的细节问题：
- 具体的诗词原文（哪一回、谁写的、完整内容）
- 具体的人物行为细节（某回某人说了什么话、做了什么事）
- 具体的物品/地点/数字

示例："第三十七回海棠诗社中，探春给自己起的别号是什么？"

==========================================================
类别二：人物关系多跳查询（需要跨回目追踪）
==========================================================
设计需要跨多个回目才能完整回答的人物关系问题：
- 某人与某人之间的间接关系链
- 某个人物在不同场景中与不同人的互动
- 某一派系/团体的成员构成

示例："王熙凤在管理贾府事务过程中，与哪些人产生过直接的利益冲突？各自因何而起？"

==========================================================
类别三：全局分析查询（需要全书视角）
==========================================================
设计需要综合全书信息进行分析的问题：
- 人物命运走向的全局分析
- 家族兴衰的原因归纳
- 主题和隐喻的总结

示例："从全书来看，贾府的经济状况是如何一步步走向入不敷出的？涉及哪些关键事件？"

==========================================================
类别四：对比分析查询（需要关联多个信息进行比较）
==========================================================
设计需要对比两个或多个实体的问题：
- 人物性格、命运的对比
- 不同事件的对比
- 不同诗作的对比

示例："薛宝钗和林黛玉在对待贾宝玉读书态度上有什么具体差异？分别在哪些场景中体现？"

每个问题请给出：
- 问题文本
- 复杂度（zero_hop/one_hop/multi_hop）
- 预期涉及的回目范围
- 参考答案要点（列出关键信息点即可）
- 为什么这个问题RAG比纯LLM更有优势（一句话）
```
